PECL Bug Tracking System

If you encounter any issues or need support, our team is here to help! Before reporting a bug, make sure to check our support channels for assistance.

Before Reporting a Bug

  1. Search Existing Bugs :

    Make sure the issue hasn't been reported before by searching our existing bugs. This helps avoid duplicate reports and speeds up the resolution process.

  2. Read Bug Reporting Tips :

    Familiarize yourself with our bug reporting tips to provide clear and helpful information. This ensures that others can easily understand and assist with the issue.

Ready to Report?

Follow these steps based on your situation:

  • For Package-Specific Bugs:

    Visit the package home page using the Browse Packages or the package search system. Report the bug directly from the package page.

  • For Non-Package Issues:

    If the bug doesn't relate to a specific PECL package, choose an appropriate category:

    • Website Problem
    • Documentation Problem

Explore Bug Statistics

For additional insights, check out our bug statistics page. It provides interesting data related to reported issues.

Thank you for contributing to the improvement of PECL! Your bug reports help us enhance the overall experience for our users.
